ANIMATION LOGIC · 动效逻辑
高置信
扫描结束即静止
线消失，框发光，点呼吸
用「停下来」表达「找到了」
→ 动效终止本身就是完成信号
中置信
慢速来回扫，到底停顿
easing 用 ease-in-out，不急促
像人在「犹豫中反复确认」
→ 节奏传达「在想」，不是在跑
低置信
快速线性单向循环
框体每隔几秒微抖一下
粒子上浮，增加不稳定感
→ 快+抖+粒子 = 系统在挣扎找答案
RECAP · 四个方案汇总
方案一
识别框状态
颜色（绿/黄/橙棕）+ 边框样式（实线/虚线）传达置信，视觉最直接，用户视线在取景框时就能感知
方案二
提示条
顶部文字引导，高置信不出现，低置信给出行动建议「靠近一点」。说人话，新用户友好
方案三
触觉反馈
咔 / 咚咚 / 嗡嗡嗡 三档震动模式，无声胜有声，用户闭眼也能感知识别质量
★ 方案四
扫描动效
静止锁定 / 慢速来回 / 快速抖动，速度和节奏本身就是信号，和识别框结合效果最佳
